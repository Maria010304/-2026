# Ifbest Enterprise — фронтенд

Интерактивный прототип корпоративной видеоплатформы для дипломной работы
«UX/UI-проектирование и front-end разработка корпоративной экосистемы для хранения,
анализа и работы с видеоконтентом Ifbest Enterprise».

Реализация главы 3 диплома. Полная front-end часть на стеке из таблицы 3.1.1.

---

## Технологический стек

| Технология | Назначение |
|---|---|
| HTML5 | Структура страниц |
| CSS3 (чистый, без препроцессоров) | Стили, две темы, адаптивность |
| JavaScript ES6+ | Логика, состояние, обработчики |
| React 18 | Компонентная архитектура |
| React Router v6 | Маршрутизация между экранами |
| Vite 5 | Сборка и dev-сервер |

Никаких UI-библиотек, CSS-фреймворков (Tailwind, Bootstrap), препроцессоров (Sass)
и сторонних иконочных пакетов. Все иконки — встроенные SVG.

## Как запустить

### 1. Установить Node.js

Нужна версия **18 или новее**. Скачать с [nodejs.org](https://nodejs.org/).
Проверка: открой терминал и набери `node --version`.

### 2. Установить зависимости

В корне проекта (`ifbest-enterprise/`) выполнить:

```bash
npm install
```

Команда скачает React, React Router и Vite. Занимает 30–60 секунд.

### 3. Запустить dev-сервер

```bash
npm run dev
```

После запуска в терминале появится адрес — обычно `http://localhost:5173`.
Откроется автоматически в браузере. Изменения в коде применяются мгновенно
(hot reload).

### 4. Сборка продакшен-версии (опционально)

```bash
npm run build
```

Готовая статика появится в папке `dist/`. Её можно открыть локально через

```bash
npm run preview
```

## Демо-аккаунты

В демо нет настоящего бэкенда — авторизация работает по списку мок-пользователей.
Пароль для всех: `password`. На странице входа есть кнопки быстрого входа.

| Email | Роль | Что видит |
|---|---|---|
| `director@ifbest.ru` | Директор | Всё: видео всех отделов, аналитику, карту решений, VCS-интеграции |
| `manager@ifbest.ru` | Руководитель отдела | Только видео отдела разработки + аналитику + карту решений |
| `employee@ifbest.ru` | Сотрудник | Только публичные видео + свои; без аналитики/карты/VCS |

Сессия сохраняется в `localStorage`, перезагрузка страницы не сбрасывает вход.
Выход — из меню в левом сайдбаре.

## Видеоплеер

В демо подключено открытое тестовое видео Big Buck Bunny через CDN Google
(`commondatastorage.googleapis.com`). Для запуска нужен интернет.

Чтобы использовать своё видео:
1. Положить файл в `public/demo.mp4`.
2. В `src/pages/VideoPlayer/VideoPlayer.jsx` заменить константу `DEMO_VIDEO_URL`
   на `'/demo.mp4'`.

Транскрипт генерируется автоматически (24 фразы равномерно распределяются
по длительности видео) — для демонстрации синхронизации этого достаточно.

## Адаптивность

Все макеты сделаны для трёх размеров:
- **Десктоп** — ≥ 1024 px, основной экран демонстрации
- **Планшет** — 600–1023 px, 2 колонки вместо 3
- **Мобильный** — < 600 px, одна колонка, компактная шапка

Переключение тёмной и светлой темы — в выдвижном сайдбаре (кнопка слева вверху).
Выбор темы сохраняется в `localStorage`.

## Что демонстрировать на защите

1. **Вход с разными ролями** — показать, как меняется навигация и доступные пункты
   меню в зависимости от роли (быстрые кнопки на странице логина).
2. **Главная** — фильтры (отдел + теги + дата), сортировка, пагинация.
3. **Плеер** — клик по фразе в транскрипте перематывает видео; при воспроизведении
   текущая фраза подсвечивается и автопрокручивается.
4. **Summary** — переключение вкладок, теги, переход на похожие видео.
5. **Карта решений** — выбрать несколько видео → "Построить" → SVG-граф решений.
6. **Загрузка** — drag&drop файла → прогресс → форма метаданных.
7. **Аналитика** — KPI, графики, активность отделов (только для руководителя/директора).
8. **VCS** — CRUD интеграций, webhook-логи (только для директора).
9. **Переключение темы** — тёмная ↔ светлая в любой момент, без перезагрузки.
